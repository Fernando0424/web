Contexto del proyecto

¿Qué es Workys y de dónde surge?

Workys nace de InnovaTec 2025, un concurso de innovación tecnológica en el ITCH2. Fui invitado a participar como único desarrollador del equipo, siendo responsable de la totalidad del desarrollo — desde la arquitectura hasta la implementación.

Stack
Flutter + Firebase

La idea detrás de Workys es similar al modelo de apps de transporte como Uber o DiDi, pero aplicado a servicios de personal para eventos: meseros, guardias de seguridad, chefs, animadores y cualquier perfil de servicio eventual.

La plataforma permite que un mismo usuario pueda tanto contratar como ofrecer servicios, cambiando de rol dentro de la misma app. Para ofrecer servicios, el sistema exige una verificación documental que valida la identidad y legalidad del trabajador.

Funcionalidades

Lo que Workys puede hacer

Cada módulo fue diseñado e implementado de forma completa en solitario durante el desarrollo del concurso.

🔐
Autenticación completa
Inicio de sesión, registro y gestión de sesión mediante Firebase Auth. Flujo seguro con validación de formularios y manejo de errores.
👤
Perfil dual
Un mismo usuario puede actuar como contratante o como trabajador. Para activar el modo trabajador se requiere la carga y verificación de documentos legales.
❤️
Servicio de favoritos
Los usuarios pueden guardar perfiles de trabajadores o servicios que les interesen para acceder rápidamente a ellos en futuras contrataciones.
⭐
Reseñas y calificaciones
Cada trabajador cuenta con un perfil de reseñas donde los contratantes pueden dejar su opinión y calificación numérica tras el servicio prestado.
🏆
Sistema de niveles
Los trabajadores acumulan experiencia y suben de nivel conforme reciben contrataciones y buenas reseñas, permitiendo a los clientes medir el rendimiento histórico.
🛡️
Panel de administrador
Usuarios con privilegios especiales (el equipo de innovación) cuentan con opciones de administración para gestionar usuarios, validar documentos y moderar contenido.
💳
Módulo de pagos
Se implementó el formulario de captura de datos de pago. El procesamiento efectivo del cobro quedó pendiente para una siguiente iteración del proyecto.
⚠️ Parcialmente implementado
📄
Verificación documental
Antes de habilitar a un usuario para ofrecer servicios, la plataforma exige la carga de documentos que acrediten su identidad y legalidad para trabajar.

Tecnología

Stack y decisiones técnicas

La elección tecnológica priorizó velocidad de desarrollo, cobertura multiplataforma y un ecosistema con amplio soporte de recursos.

🐦

Flutter

UI multiplataforma desde un solo codebase

🔥

Firebase Auth

Autenticación segura y gestión de sesiones

🗄️

Firestore

Base de datos en tiempo real NoSQL

📦

Firebase Storage

Almacenamiento de documentos e imágenes

🌐

Firebase Hosting

Deploy de la versión web

Opté por Flutter + Firebase por la experiencia previa que manejaba con estas tecnologías, la facilidad para desarrollar en solitario cubriendo tres plataformas simultáneamente (Android, iOS y Web), y la amplia disponibilidad de documentación y herramientas de apoyo.

Firebase permitió construir el backend funcional (autenticación, base de datos, storage y hosting) sin necesidad de un servidor propio, lo que fue clave dado el tiempo limitado del concurso y el hecho de trabajar solo.

Limitaciones conocidas

Lo que quedó pendiente

⚠️
Módulo de pagos parcial: Se implementó el flujo completo de captura de datos de pago (formulario, validación de campos), pero el procesamiento efectivo del cobro no fue integrado. La intención era conectar con una pasarela de pago en una siguiente versión del proyecto, fuera del alcance del concurso.
