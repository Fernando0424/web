Sistema de Gestión Académica

Sistema de escritorio en Java puro para la administración completa de alumnos, docentes, cursos y clases. El proyecto en el que más tiempo y dedicación he invertido de todos mis sistemas CRUD a lo largo de la carrera.

Contexto del proyecto

El proyecto más trabajado de su tipo

A lo largo de la carrera he desarrollado múltiples sistemas CRUD. Este es el que más esfuerzo y dedicación requirió. No es solo un sistema de registro: es una plataforma de gestión completa con cuatro entidades relacionadas, módulo de autenticación y arquitectura bien estructurada.

La decisión de usar Java puro sin frameworks externos y seguir el patrón MVC (Modelo–Vista–Controlador) fue intencional: el objetivo era demostrar dominio de los fundamentos de la programación orientada a objetos y de la arquitectura de software, sin apoyarme en atajos.

Funcionalidades

¿Qué puede hacer el sistema?

📋
Tablas con listado completo
Cada entidad tiene su vista de tabla donde se muestra el listado completo con todos los campos, con opción de seleccionar un registro para editarlo o eliminarlo.
➕
CRUD completo por entidad
Crear, leer, actualizar y eliminar registros para cada una de las cuatro entidades (alumnos, docentes, cursos y clases) de forma independiente.
🔐
Sistema de autenticación
Pantalla de inicio de sesión con validación de credenciales. Solo usuarios autenticados pueden acceder al panel de administración del sistema.
💾
Persistencia local
Los datos se almacenan en un repositorio local. La información persiste entre sesiones sin depender de una base de datos externa ni conexión a internet.
Arquitectura

Patrón MVC aplicado

El sistema está construido siguiendo el patrón Modelo–Vista–Controlador de forma estricta, separando responsabilidades en tres capas bien definidas.

🗃️
Modelo
Clases Java que representan las entidades (Alumno, Docente, Curso, Clase) y los repositorios locales responsables de la persistencia y consulta de datos.
🖥️
Vista
Interfaces gráficas construidas en Java (Swing) para cada módulo: tablas, formularios de alta/edición y pantalla de login. La vista solo muestra datos, no los procesa.
⚙️
Controlador
Clases controladoras que reciben las acciones del usuario desde la vista, invocan la lógica del modelo y actualizan la vista con el resultado. Toda la lógica de negocio vive aquí.
💡
Usar MVC en Java puro, sin frameworks como Spring o JavaFX, implicó definir manualmente la comunicación entre capas. Esto fue un ejercicio valioso para entender por qué los frameworks existen y qué problema resuelven, antes de depender de ellos.
